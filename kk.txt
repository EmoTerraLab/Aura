Es un placer saludarte. Como experto en el protocolo de Aragón (Resolución de 19 de octubre de 2018) y arquitecto de Aura PDP, he diseñado la estructura de datos necesaria para transformar la gestión
  de tickets en un sistema de cumplimiento legal riguroso.

  Para implementar la máquina de estados y los Anexos oficiales, no modificaremos la tabla reports base para mantener la integridad del núcleo SaaS, sino que la extenderemos mediante una relación 1:1
  con una nueva tabla de control y tablas satélite para cada hito del protocolo.

  Aquí tienes las migraciones SQLite diseñadas siguiendo los estándares de Aura PDP v2.0.0:

  1. Control de Flujo y Máquina de Estados (Extensión de Report)
  Esta tabla vincula un reporte con el ciclo de vida legal de Aragón y gestiona la decisión inicial (Anexo I-b).

    1 -- Gestión del expediente oficial de Aragón
    2 CREATE TABLE IF NOT EXISTS aragon_protocol_cases (
    3     id INTEGER PRIMARY KEY AUTOINCREMENT,
    4     report_id INTEGER NOT NULL UNIQUE,          -- Vinculación con la tabla reports base
    5     protocol_code VARCHAR(50) UNIQUE,           -- Código correlativo interno (ej. PROT-2024-001)
    6     current_state VARCHAR(50) DEFAULT 'comunicacion_recibida',
    7     
    8     -- Decisión de Inicio (Anexo I-b)
    9     is_started BOOLEAN DEFAULT 0,
   10     decision_date DATETIME,
   11     decision_reason TEXT,                       -- Justificación de la resolución del Director/a
   12     
   13     -- Comunicaciones Obligatorias
   14     notified_inspection_at DATETIME,            -- Fecha envío Anexos I-a/I-b a Inspección
   15     notified_eoe_convivencia_at DATETIME,       -- Fecha envío Anexos I-a/I-b a equipoconvivencia@aragon.es
   16     
   17     -- Control de Tiempos
   18     school_days_counter INTEGER DEFAULT 0,      -- Contador de días lectivos (límite día 20/22)
   19     deadline_valuation DATETIME,                -- Calculado: +20 días lectivos
   20     
   21     created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
   22     updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
   23     FOREIGN KEY (report_id) REFERENCES reports(id) ON DELETE CASCADE

   25
   26 CREATE INDEX IF NOT EXISTS idx_aragon_case_state ON aragon_protocol_cases(current_state);
   27 CREATE INDEX IF NOT EXISTS idx_aragon_case_report ON aragon_protocol_cases(report_id);

  Notas técnicas para la implementación en Aura:

   1. Seguridad RGPD: El Anexo VIII está marcado como is_confidential_confirmed. En el controlador ProtocolWorkflowController, debes asegurar que los campos de esta tabla solo sean accesibles para el
      rol admin (Director) y los users vinculados en la tabla aragon_annex_iii_team.
   2. Violencia Sexual: Si en aragon_annex_v_interviews se marca abuse_type_sexual, la interfaz debe disparar un Helper que oculte los campos de texto libre de "Descripción de hechos" para cumplir con
      la norma de "Solo escuchar, no interrogar".
   3. Índices: He añadido índices en case_id para todas las tablas dependientes, lo que garantiza que la generación del informe consolidado (PDF) sea instantánea incluso en bases de datos SQLite con
      años de histórico.
